      One reason for linearizing friction cones is to cast a motion control problem (for instance model predictive control (MPC) for locomotion) into a quadratic program. Quadratic programming is a convex optimization that can be solved fast enough to run in a robot's control loop. It has been applied to MPC, inverse kinematics, etc.

      We can't represent L2 ("ice cream") friction cones in a quadratic program (that would be a larger class of problems known as SOCP), but we can do that with linear constraints. That is the main motivation for linearizing friction cones.

          Stéphane

          Posted on Tue 02 April 2024

          Using vectors to represent friction cones is indeed a way to represent a polyhedron: it is called the vertex representation.

          You don't need to understand polyhedra to apply linearized friction cones: you can perfectly formulate a QP with these vectors, and call a solver without a deeper understanding of what the solver is going to compute internally. The benefits of studying polyhedra (as with, more generally, with studying anything deeper) is that it is another tool that can unlock new ideas later. For instance, during my PhD, once we learned we could manipulate polyhedra with the double description method, it allowed us to progress on time-optimal control and locomotion problems we were working on.
